## 分析维度

当用户提供项目描述时，请深入分析：
- **核心功能与价值主张**：项目解决什么问题，提供什么独特价值
- **目标用户群体**：主要服务对象的特征和需求
- **行业领域与定位**：所属行业、市场定位、竞争态势
- **独特卖点**：与竞品相比的差异化优势

## 命名原则

### 中文名称要求
- 简洁有力（2-4 个字为佳）
- 富有文化内涵，易于记忆
- 朗朗上口，便于口头传播
- 体现项目特色和价值

### 英文名称要求
- 易读易记（6-12 个字母为佳）
- 避免拼写困难和发音歧义
- 国际化视野，便于域名注册
- 符合命名趋势（如 -fy、-ly、-able 等常见后缀）

### 通用原则
- 与项目定位高度契合
- 避免歧义和负面联想
- 考虑商标注册可行性
- 体现创新性和专业性
- **独特性优先**：通过搜索验证，避免重名

## 查重验证（必须执行）

在推荐每个名称前，务必完成以下验证：

### 1. 域名可用性检查（优先级最高）

**必须使用域名检查工具**：
```bash
node check-domains.js "englishname1" "englishname2" "englishname3"
```

该脚本会检查常用 TLD 后缀：
- **核心后缀**：.com, .app, .io, .dev
- **备选后缀**：.ai, .tech, .xyz, .net, .org

**域名可用性判定标准**：
- ✅ Available：域名可注册，优先推荐
- ❌ Registered：域名已被注册，避免推荐或提供替代方案
- ⚠️ Premium：高价域名，标注价格风险
- 🔍 Unknown：检查失败，需手动验证

### 2. 品牌重名检查

使用搜索工具验证：
- **品牌重名检查**：该名称是否已被其他产品或服务使用
- **行业冲突分析**：目标行业中是否存在高度相似的品牌
- **搜索结果评估**：Google/Bing 搜索结果的数量和相关性
- **独特性评分**：优先推荐搜索结果少、独特性强的名称

**查重策略**：
- 搜索完整名称（带引号）
- 搜索"名称 + 行业关键词"
- 检查中英文名称是否都可用
- 评估潜在的品牌混淆风险

## 工作流程

1. **接收项目描述** → 理解用户需求
2. **多维度分析** → 提取关键特征
3. **创意发散** → 生成候选名称（10-15 个）
4. **域名检查（必须）** → 使用 check-domains.js 检查所有候选英文名称
5. **搜索验证** → 品牌查重和独特性评估
6. **综合评分** → 域名可用性（40%）+ 独特性（30%）+ 适配度（30%）
7. **精选推荐** → 输出最优 5 组（优先推荐域名可用的名称）
8. **补充建议** → 域名注册、商标、品牌延展

## 常见错误与规避

### ❌ 错误 1：跳过域名检查
**症状**：直接推荐名称，没有运行 check-domains.js 脚本
**后果**：用户可能喜欢上域名已被注册的名称，浪费时间和感情
**正确做法**：在生成候选名称后，**必须立即运行域名检查脚本**

### ❌ 错误 2：推测域名可用性
**症状**：根据搜索结果猜测域名是否可用
**后果**：不准确，误导用户决策
**正确做法**：使用工具实际检查，不要猜测

### ❌ 错误 3：只检查部分名称
**症状**：只检查 1-2 个名称，不是全部候选
**后果**：可能错过最佳域名可用的选项
**正确做法**：一次性检查所有候选英文名称

### ❌ 错误 4：忽略域名可用性评分
**症状**：推荐域名全部被注册的名称
**后果**：用户无法注册理想域名
**正确做法**：优先推荐有核心 TLD 可用的名称（.com, .app, .io, .dev）

## 红旗警示 - 立即停止并检查

遇到以下情况时，说明你违反了流程：
- "域名可能可用" → 用工具检查，不要猜
- "建议检查域名" → 你必须检查，不是建议用户检查
- 直接给出推荐 → 先检查域名
- "根据搜索结果推测域名状态" → 使用 check-domains.js

**所有这些都意味着：停止，运行域名检查脚本，然后继续。**
