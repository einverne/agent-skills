# 视频脚本结构指南

本文档定义了 einverne 技术频道视频脚本的标准结构和组成部分。

## 标准视频时长

- **目标时长**：6-10 分钟
- **建议分段**：每 2-3 分钟一个主要段落，保持观众注意力

## 脚本标准结构

### 1. 开场吸引段 (0:00-0:20, ~20秒)

> **核心原则：前 20 秒决定观众是否继续看！**

**必须包含元素：**
- 吸引点（效果预览/痛点共鸣/价值承诺，选一种）
- 今日话题介绍（1-2 句话）

**示例：**
```
[展示最终效果] 是不是很酷？今天我就带大家一步步实现这个效果。
今天我们用 Docker Compose，10 分钟搞定数据库、缓存、消息队列的整套环境配置。
```

### 2. 概念介绍段 (0:20-2:00, ~1.5分钟)

**风格：技术分享（轻松、对话式）**

**必须包含元素：**
- 核心概念/技术背景
- 为什么需要这个技术/解决什么问题
- 适用场景和常见误区
- 与相关技术的对比（如适用）

**镜头提示：**
- 主镜头：轻松交流感，可以有手势
- B-roll：概念图、架构图、对比图表、官网截图

**呈现技巧：**
- 用类比和实际例子解释抽象概念
- 避免术语堆砌，用通俗语言
- 可以提及个人经验或踩过的坑

### 3. 准备工作段 (2:00-3:00, ~1分钟)

**必须包含元素：**
- 前置条件/环境要求
- 需要的工具和版本
- 推荐的学习资源（简要）
- 项目结构预览

**镜头提示：**
- 主镜头：保持专业，开始转向教学模式
- B-roll：工具安装界面、官方文档页面、项目目录结构

**要点：**
- 清单式列出所有依赖
- 提供版本号和兼容性信息
- 对新手友好的详细程度

### 4. 实战教程段 (3:00-8:00, ~5分钟)

**风格：正式教学（结构化、步骤清晰）**

**结构模式：**
```
步骤 1: [操作名称]
- 为什么这样做
- 具体操作
- 预期结果
- 常见错误

步骤 2: [操作名称]
...
```

**必须包含元素：**
- 分步骤编号（步骤 1、步骤 2...）
- 每步的目的和预期结果
- 代码/配置文件的逐行解释
- 常见错误和解决方案
- 检验点（如何确认成功）

**镜头提示：**
- **主要内容：屏幕录制**
  - 实时操作演示
  - 代码编写过程
  - 终端命令执行
- **辅助镜头：画中画人像**（可选）
  - 在屏幕右下角或右上角
  - 关键解释时出现
- B-roll：
  - 文档页面特写
  - 配置文件对比
  - 运行结果展示

**节奏控制：**
- 每个步骤 1-2 分钟
- 复杂步骤可拆分成子步骤
- 在关键点暂停，强调要点

**语言技巧：**
- 使用"现在我们..."、"接下来..."、"注意这里..."等过渡语
- 代码解释：先说做什么，再说怎么做，最后说为什么
- 错误处理：先预防，后解决

### 5. 验证测试段 (8:00-9:00, ~1分钟)

**必须包含元素：**
- 完整的功能验证流程
- 测试命令/操作
- 预期输出展示
- 故障排除快速指南

**镜头提示：**
- 屏幕录制：完整演示测试过程
- B-roll：成功/失败的典型输出对比

### 6. 扩展延伸段 (可选，融入结尾)

**必须包含元素：**
- 进阶使用技巧
- 性能优化建议
- 最佳实践推荐
- 相关工具/技术推荐

**镜头提示：**
- 主镜头：回到面对镜头，分享经验
- B-roll：相关资源截图、工具界面

### 7. 结尾段 (9:00-10:00, ~1分钟)

**必须包含元素：**
- 内容总结（1-2 句话）
- 后续学习建议
- 互动引导（可选）
- 预告下期内容（可选）

## 时间轴标记规范

在脚本中使用标准化的时间轴标记：

```
[00:00-01:00] 段落标题
```

## 段落过渡技巧

**常用过渡语：**
- "理解了这些概念之后，我们来看看实际操作"
- "在开始之前，我们需要准备以下工具"
- "接下来是最关键的部分"
- "完成这一步之后，我们来验证一下效果"
- "掌握了基础操作，我再分享几个进阶技巧"

## 长度调整策略

**如果内容过长（>10分钟）：**
- 简化概念介绍，聚焦核心
- 减少代码逐行解释，只讲关键部分
- 将扩展内容移到评论区或后续视频

**如果内容过短（<6分钟）：**
- 增加实际案例和应用场景
- 添加常见问题和解决方案
- 补充最佳实践和优化技巧
